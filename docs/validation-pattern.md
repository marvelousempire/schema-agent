# Validation Pattern

## Purpose

The Validation Pattern explains how an agent checks its work.

## Statuses

```text
Complete
Needs Review
Missing Information
Blocked
```

## Validation Questions

- Did the agent understand the request?
- Did the agent update the right file?
- Did the agent preserve intent?
- Did the agent update ledgers when needed?
- Did the agent add journals when needed?
- Did the agent report blocked work honestly?

## Rule

Validation should be honest, not decorative.
