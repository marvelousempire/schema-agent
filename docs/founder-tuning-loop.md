# Founder Tuning Loop

## Purpose

The Founder Tuning Loop explains how live corrections become agent behavior.

## Core Idea

When the founder gives a correction, preference, naming rule, structure rule, tone rule, or workflow rule, the agent should decide whether it should become part of the schema.

## Loop

```text
Founder instruction
→ Identify reusable meaning
→ Decide where it belongs
→ Update rule, doc, ticket, template, or journal
→ Validate the change
→ Report what changed
```

## Examples

A correction can become:

- A rule
- A feature ticket
- A README update
- A prompt file
- A template
- A journal entry
- A validation checklist item

## Rule

Do not let useful nuance disappear.

If a detail improves repeatable behavior, capture it.
