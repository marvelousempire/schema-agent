# Validation

## Purpose

This file explains how the agent checks its work.

## Statuses

```text
Complete
Needs Review
Missing Information
```

## Checklist

- The request was understood.
- The right file was created or updated.
- The work matches the repo structure.
- The feature ledger was updated when needed.
- The journal was updated when needed.
- The final report is honest.

## Rule

Do not call unfinished work complete.
