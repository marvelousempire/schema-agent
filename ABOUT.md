# About Schema Agent

Schema Agent is a reusable blueprint for creating AI agents.

It exists because good agents need more than a clever prompt. They need structure, memory, rules, validation, feature tickets, journals, templates, and a clear place to grow.

## Why This Exists

When an agent is built through live conversation, important decisions appear in small moments.

A phrase becomes a rule.

A correction becomes a behavior.

A repeated action becomes a template.

A missing folder becomes a repo standard.

Schema Agent captures that process so it can be repeated safely.

## What It Does

Schema Agent defines:

- Agent repo structure
- Agent README pattern
- Agent ABOUT pattern
- Agent PRD pattern
- System prompt folder pattern
- Firing order pattern
- Rule stack pattern
- Feature ticket pattern
- Journal pattern
- Template pattern
- Validation pattern
- Release ledger pattern
- Handoff sheet pattern

## Design Philosophy

Agents should be built like living systems.

Each part should be small enough to understand and strong enough to reuse.

The schema should preserve founder intent while staying clean enough for another AI or developer to follow.

## Source Example

The first working example of this schema is:

```text

```

Schema Agent extracts the reusable agent-building method from that work.
